# Installation & Setup

This guide covers everything you need to know about setting up, configuring, and customizing the Mikunki Shapshico festival website.

## File Structure

The project is intentionally minimal with just three core files:

```
source/
├── index.html      # Complete HTML structure (579 lines)
├── script.js       # All JavaScript functionality (1,102 lines)
├── styles.css      # Complete styling (2,200+ lines)
└── README.md       # Project documentation
```

<Note>
  No build process, no dependencies, no node_modules. Just pure HTML, CSS, and JavaScript!
</Note>

## Performance Optimization

### Image Optimization

<Steps>
  <Step title="Use WebP format">
    Convert images to WebP for 25-35% smaller file sizes:
    
    ```html
    <picture>
      <source srcset="image.webp" type="image/webp">
      <img src="image.jpg" alt="Fallback">
    </picture>
    ```
  </Step>

  <Step title="Implement lazy loading">
    Already implemented in gallery (line 446):
    
    ```html
    <img src="..." loading="lazy" />
    ```
  </Step>

  <Step title="Optimize video files">
    Fire videos are already optimized via Cloudinary:
    - Auto format selection (`f_auto`)
    - Quality optimization (`q_auto:low`)
    - Width limiting (`w_640` or `w_720`)
    - Video codec optimization (`vc_auto`)
  </Step>
</Steps>

### Reduce Motion

The site respects `prefers-reduced-motion`:

```javascript source/script.js
if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
  return;  // Skip particle animations
}
```

To disable all animations, add to CSS:

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### Mobile Optimization

Particle count auto-reduces on mobile (line 39):

```javascript
const isMobile = window.matchMedia('(pointer: coarse)').matches;
const count = isMobile ? 10 : CONFIG.PARTICLE_COUNT;
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Countdown shows 00:00:00">
    Check that `FESTIVAL_DATE` is in the future:
    
    ```javascript
    FESTIVAL_DATE: new Date("2025-11-18T10:00:00"),
    ```
    
    Ensure the timezone matches your server's timezone.
  </Accordion>

  <Accordion title="Chat points not accumulating">
    Points are awarded every 30 seconds (line 815):
    
    ```javascript
    setInterval(() => {
      if (!currentUser || document.hidden) return;
      currentUser.points = Math.min(9999, (currentUser.points || 0) + 2);
    }, 30000);
    ```
    
    Make sure the tab is active (not hidden).
  </Accordion>

  <Accordion title="Player and chat heights don't match">
    The layout sync function runs on resize (lines 239-280). Try:
    - Hard refresh (Ctrl+Shift+R)
    - Resize browser window to trigger sync
    - Check console for JavaScript errors
  </Accordion>

  <Accordion title="Animations are janky">
    Performance optimizations to try:
    - Reduce `PARTICLE_COUNT` to 10-15
    - Disable parallax effect (comment out lines 837-861)
    - Use `will-change` CSS property sparingly
  </Accordion>
</AccordionGroup>
