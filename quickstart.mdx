# Quick Start Guide

Get the Mikunki Shapshico festival site up and running on your local machine in just a few steps. This is a static HTML site with no build process required!

## Prerequisites

You only need:
- A modern web browser (Chrome, Firefox, Safari, or Edge)
- A local web server (optional but recommended)

<Note>
  While you can open `index.html` directly in a browser, using a local server prevents CORS issues with external resources.
</Note>

## Running the Site

<Steps>
  <Step title="Navigate to the source directory">
    Open your terminal and navigate to the project folder:
    
    ```bash
    cd ~/workspace/source
    ```
  </Step>

  <Step title="Start a local web server">
    Choose one of these methods based on what you have installed:

    <CodeGroup>
      ```bash Python 3
      python3 -m http.server 8000
      ```

      ```bash Python 2
      python -m SimpleHTTPServer 8000
      ```

      ```bash Node.js (npx)
      npx serve .
      ```

      ```bash PHP
      php -S localhost:8000
      ```
    </CodeGroup>

    <Note>
      If you don't have any of these installed, you can also open `index.html` directly by double-clicking it.
    </Note>
  </Step>

  <Step title="Open in your browser">
    Navigate to the local server URL:
    
    ```
    http://localhost:8000
    ```

    You should see the Mikunki Shapshico festival homepage with:
    - Animated hero section with countdown timer
    - Floating particle effects
    - Festival program timeline
    - Live streaming section (waiting screen if not live)
    - Interactive chat interface
    - Photo gallery
  </Step>
</Steps>

## What You'll See

### Hero Section
The landing section features:
- Festival logo and wordmark
- Animated subtitle with typewriter effect (source/script.js:867-888)
- Real-time countdown to November 18, 2025 (source/script.js:161-200)
- CTA buttons linking to festival info and live stream
- Fire video overlay with parallax scrolling

### Navigation
- Fixed header that follows you as you scroll
- Mobile-responsive hamburger menu
- Smooth scroll to sections
- Active link highlighting based on visible section

### Live Streaming Area
<Warning>
  The stream shows a waiting screen by default. To test with a live video, update the configuration (see below).
</Warning>

The live section includes:
- 16:9 video player with Vimeo integration
- Channel information panel
- Share button for social sharing
- Real-time viewer counter
- Interactive chat system

### Chat System
Click "Iniciar sesión" to create a test account:
1. Enter a username
2. Pick a color
3. Start chatting!

Chat features:
- Point-based economy (earn 2 points every 30 seconds)
- Superchat tiers: Bronze (40pts), Silver (80pts), Gold (120pts)
- Quick emoji reactions
- Persistent sessions via localStorage

## Testing the Live Stream

To test with an actual video:

<Steps>
  <Step title="Open script.js">
    Edit the configuration at the top of `source/script.js`:
    
    ```javascript source/script.js
    const CONFIG = {
      STREAM_URL: "https://player.vimeo.com/video/76979871?autoplay=1&muted=1&title=0&byline=0&portrait=0",
      IS_LIVE: true,
      FESTIVAL_DATE: new Date("2025-11-18T10:00:00"),
      PARTICLE_COUNT: 25
    };
    ```
  </Step>

  <Step title="Update the Vimeo URL">
    Replace the video ID (76979871) with your own Vimeo video:
    
    ```javascript
    STREAM_URL: "https://player.vimeo.com/video/YOUR_VIDEO_ID?autoplay=1&muted=1&title=0&byline=0&portrait=0"
    ```
  </Step>

  <Step title="Refresh the page">
    Reload your browser to see the video player instead of the waiting screen.
  </Step>
</Steps>

## Quick Configuration

### Change Festival Date

Update the countdown timer date in `source/script.js:13`:

```javascript
FESTIVAL_DATE: new Date("2025-11-18T10:00:00"),  // Change this date
```

### Adjust Particle Count

For better performance on lower-end devices:

```javascript
PARTICLE_COUNT: 25  // Reduce to 10-15 for slower devices
```

<Note>
  The script automatically reduces particles to 10 on mobile devices (source/script.js:39).
</Note>

### Toggle Live Status

Control whether the stream shows or displays waiting screen:

```javascript
IS_LIVE: true,  // Set to false to show waiting screen
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Fire video not playing">
    The fire video overlay uses autoplay with mute. Some browsers block autoplay:
    - Ensure audio is muted (it is by default)
    - Try a different browser
    - Check the Cloudinary URL is accessible
    
    Video playback rate is set to 0.3x for a slow burn effect (source/script.js:829).
  </Accordion>

  <Accordion title="Fonts not loading">
    The site uses Google Fonts (Bebas Neue and Poppins):
    - Check your internet connection
    - Look for preconnect warnings in DevTools
    - Fonts are preloaded with `display=swap` for better UX
  </Accordion>

  <Accordion title="Icons not showing">
    Font Awesome 6.5.1 is loaded from CDN:
    - Verify CDN is accessible: `cdnjs.cloudflare.com`
    - Check browser console for 404 errors
    - Icons are referenced with classes like `fa-solid fa-fire`
  </Accordion>

  <Accordion title="Chat not persisting">
    Chat uses localStorage for session persistence:
    - Check if localStorage is enabled in your browser
    - Try logging out and back in
    - Clear localStorage: `localStorage.removeItem('mikunki_chat_user')`
  </Accordion>
</AccordionGroup>
